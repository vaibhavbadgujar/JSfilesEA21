Q1: What is execution context?

ANS:
Each and every browser having Javascript engine.For example,Google is having V8 engine and Mozilla firefox having Spider Monkey.Javascript engine uses a call stack and create a special environment to handle the execution of javascript code.This environment is called execution context.

There are 2 kinds of execution context:
1.Global execution context
2.Function execution context.

There are 2 phases of each execution context
A: Creation phase
B: Execution phase

# Under Creation phase, Memory allocation ,variable declaration and function declaration are have.

# Under Execution phase, Javascript code is executed line by line..

Q2: WHat is event loop and call stack?

ANS:

Event loop:

Event loop is constantly running process that monitors both callback queue and call stack.All Web API's like settimeout and set interval are not directly inside the execution context.Fastly,they move to the callback queue.Event loop checks that call stack is empty or not.If call stack is empty,then the function pushed from callback queue to the call stack.

Call stack:

Call stack is the mechanism for an interpreter to keep track of its place in a script with calls multiple function - what function is being currently run and what function are called from within the function.And it stores global and function Execution context..

Q7: What is callback hell?

ANS :

Callback hell is the pattern with mutiple callback functions which makes code hard to read and debug when dealing with asynchronous operations.
