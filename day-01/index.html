Q1: Difference between "==" and "===" operators.

ANS:--
Both are comparsion operators and return boolen values."==" is used to compare values but "===" is used to compare value and datatype both.

                   var a = 10;
                   var b ="10";
                   console.log(a==b); 
                   console.log(a===b);

Explanation: console.log(a==b)give result true because the value i.e.10 is same in a and b.But console.log(a===b) give result false because type of a is number and type of b is string.

Q2: What are the differences between var,let and const?

ANS:

Var:

  1. Var can be redeclared and reintialised.
  2. Var perform hoisting.
  3. Var is global scope and function scope.
  4. Used introduce before ES6

Let:

  1. Let can be reintialised but cannot be redeclared.
  2. let not perform hoisting
  3. It is under temporal dead zone.
  4. Introduce in ES6

Const:

  1. Const cannot be redeclared and reintialised
  2. Const not perform hoisting
  3. It is under temporal dead zone.
  4. Introduce in ES6

Q3: What is hoisting??

ANS:
Hoisting is the javascript mechanism where variable and function declaration are moved to top of their scope before the code execution.Remember that Javascript only hoists declaration , not initialisation..

               console.log(a);
                var a=10;

Explanation: In this, var move to top of their scope but Javascript only hoist declaration.In this initialisation occur after console.log. So, it give error.

Q4: What is a Temporal dead Zone?

ANS:
Temporal dead zone is the behaviour in Javascript that occurs when declaring a variable with let and const keyword but not with var keyword.A let or const keyword is said to be in TDZ from start of the block until the code execution where the variable is declared but not initalised.

Q6: What are pure functions??

                    console.log(a*b);
